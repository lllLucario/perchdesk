# Patterns

## Empty States

- Empty states should explain what the user can do next
- They should feel useful, not apologetic
- Use brand tone sparingly and keep copy concise

## Loading States

- Prefer structural skeletons over spinners when layout is known
- Loading UI should preserve the calm feel of the product
- Avoid flashy shimmer treatments

## Error States

- Errors should be clear, direct, and actionable
- Reserve red for actual problems, not general emphasis
- Keep the layout stable when errors appear

## Responsive Behavior

- Mobile should adapt the flow, not remove important actions
- Dense booking actions should stack cleanly
- Discovery surfaces can simplify composition on small screens, but must keep hierarchy intact

## Accessibility

- Color is never the only signal for seat or booking state
- Focus states must stay obvious in both light and dark themes
- Typography and contrast should support long reading and repeated use

## Content Style

- Use direct language
- Avoid repeating labels the user can already see
- Booking-critical text should prefer clarity over cleverness
- Descriptive surfaces may lean more editorial, but never at the expense of task clarity
