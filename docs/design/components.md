# Components

## Principles

Reusable components should feel quiet, stable, and intentional. They should not compete with the content they contain.

## Buttons

- Only one primary CTA per region should use full Moss Brand treatment
- Secondary actions should typically use Soft Reed on light surfaces or Deep Forest on dark surfaces
- Button hierarchy must be obvious without increasing saturation everywhere

## Inputs

- Inputs should prioritize legibility over decoration
- Focus state uses accessible blue focus treatment rather than bright green glow
- Error and validation styling should stay restrained and readable

## Cards & Panels

- Cards should use spacing and typography for hierarchy before relying on color fills
- Avoid card-within-card nesting unless the inner container has a clear semantic role
- Booking summary panels should feel stable and operational, not promotional

## Navigation

- Navigation should stay quiet
- Active state may use green emphasis, but not every nav item needs chromatic styling
- The brand title can carry more personality than the rest of the nav

## Tags, Chips, and Status Labels

- Use muted fills and low-noise shapes
- Green states should indicate positive or selected meaning, not just decoration
- Occupied, unavailable, and maintenance states should be distinct without creating a red-heavy interface

## Booking-Specific Components

The following components need strict visual hierarchy:
- seat status indicators
- booking summary cards
- time-slot selectors
- filter bars
- space discovery cards

On these components:
- status clarity comes before brand expression
- brand expression should support, not override, operational meaning
